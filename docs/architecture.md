# Architecture Notes

## Module Layout

- `lib/crypto`: cryptographic primitives and key derivation helpers
- `lib/format`: KDBX parsing/serialization model and merge logic
- `lib/utils`: binary, XML, and helper utilities
- `lib/index.ts`: public API surface and named exports

## Build and Distribution Strategy

- npm package entry points are generated by `tsup` into `dist/pkg`:
  - `index.mjs` for ESM consumers
  - `index.cjs` for CommonJS consumers
  - `index.d.ts` for TypeScript types
- Browser script compatibility is preserved with Webpack UMD bundles:
  - `dist/kdbxweb.js`
  - `dist/kdbxweb.min.js`

## Argon2 Integration Point

Argon2 implementation is intentionally external. Consumers register it through:

```ts
CryptoEngine.setArgon2Impl((password, salt, memory, iterations, length, parallelism, type, version) => {
    // return Promise<Uint8Array>
});
```

This keeps runtime flexibility for browser, Node, and wasm/native backends without hardcoding one Argon2 dependency into the package.
